# Agent Guide: omni-scfmt Repository

## Project Overview

**omni-scfmt** is a single-cell data format benchmark investigating:
1. Cross-tool format penalties (Seurat reading h5ad, scanpy reading RDS/h5seurat)
2. Head-to-head converter comparison (scx vs anndataR)

This is an **OmniBenchmark** project using containerized profiling (podman + denet + eBPF).

## Module Structure Pattern

Each module follows this structure:

```
modules/<stage>/<module_name>/
├── profile              # Entrypoint: runs with denet+podman profiling
├── run.R or run.py      # Actual implementation
└── (optional) dev       # Bare execution for development/testing
```

- **profile**: Production entrypoint listed in `omnibenchmark.yaml`, runs with inner denet profiling
- **run.{R,py}**: The actual work (load data, convert, analyze)
- All modules use CLI args: `--output_dir`, `--name`, and input-specific args

## Key Technologies

### OmniBenchmark
- Framework for reproducible benchmarks
- Generates Snakemake workflows from `benchmark.yaml`
- Command: `pixi run benchmark` (alias for `ob run benchmark.yaml --dirty --unpinned`)

### Profiling Stack
- **Outer layer**: `denet` + `podman` with cgroup limits + eBPF syscall tracing
- **Inner layer**: `denet` inside container for finer-grained per-process CPU/RSS sampling
- Outputs: `out/.denet/*.json` (outer) + `<output_dir>/*.inner_denet.jsonl` (inner)

### Formats
- **h5ad**: scanpy/AnnData native (HDF5-based)
- **h5seurat**: Seurat HDF5 format (via SeuratDisk)
- **rds**: R serialization (SCE or Seurat objects)

### Converters
- **scx**: Rust CLI (`/home/b/phd/scx`), handles h5ad ↔ h5seurat
- **anndataR**: R/Bioconductor package, handles h5ad ↔ rds (SCE/Seurat)

## Workflow Commands

```bash
# Build container
podman build -t localhost/scfmt:latest .

# Run full benchmark
pixi run benchmark

# Run specific module in dev mode (example)
cd modules/method/scanpy_inmemory_h5ad
./dev --h5ad /path/to/data.h5ad --output_dir /tmp/test --name test_run

# Check profiling outputs
ls out/.denet/          # Outer denet JSON
ls out/.logs/           # Execution logs
```

## Important Files & Conventions

### benchmark.yaml
- Defines 3 stages: `data` → `convert_*` → `method`
- Each stage declares `inputs` and `outputs`
- Resource limits: `cores`, `mem_mb` per module

### Stage Timing Convention
All modules use `stage()` helper (from `scripts/stage.{R,py}`) to emit timing markers:

```r
# R example
stage("load", { obj <- readRDS(args$rds) })
stage("access", { subset <- obj[sample_idx, ] })
stage("write", { saveRDS(obj, output_path) })
```

```python
# Python example
with stage("load"):
    adata = sc.read_h5ad(h5ad_path)
with stage("access"):
    subset = adata[idx, :]
with stage("write"):
    adata.write_h5ad(output_path)
```

These emit structured events to `omnibench-events.jsonl` and produce `{name}.timing.tsv`.

### Expected Outputs

**Data modules**: `{dataset}.h5ad`

**Convert modules**: Target format + timing
- scx modules: `.h5seurat` or `.h5ad` + `.prov.json`
- anndataR modules: `.rds` files

**Method modules**: Analysis results
- R/Seurat: `{name}.timing.tsv` + `{name}.results.rds` + `{name}.scratch.{ext}`
- Python/scanpy: `{name}.timing.tsv` + `{name}.access.tsv` + `{name}.scratch.h5ad`

### Environment Variables
- `SCFMT_ROOT`: Project root, used to locate shared scripts
- Set automatically by module `profile` scripts

## Common Tasks

### Adding a New Module
1. Create directory in appropriate stage: `modules/<stage>/<new_module>/`
2. Create `run.{R,py}` with CLI arg parsing and stage() markers
3. Create `profile` script wrapper (see existing modules for template)
4. Add entry to `omnibenchmark.yaml` entrypoints
5. Add module to appropriate stage in `benchmark.yaml`

### Debugging Failed Modules
1. Check `out/.logs/<rule_name>.log` for execution logs
2. Check `out/.denet/<rule_name>.json` for profiling data
3. Verify expected output files exist in `out/data/<dataset>/...`
4. Run in dev mode: `cd modules/<stage>/<module> && ./run.{R,py} --args`

### Understanding Data Flow
```
data/pbmc3k → pbmc3k.h5ad
            ↓
    ┌───────┼────────┐
    ↓       ↓        ↓
h5seurat  rds_sce  rds_seurat  (convert stage)
    ↓       ↓        ↓
  method modules read different formats
  (Seurat native vs foreign, scanpy native vs foreign)
```

## Notes for Agents

1. **Logs are mostly JSON**: The `.log` files contain extensive denet profiling JSON. To find errors, look for non-JSON lines or the end of the file.

2. **Two profiling layers**: Don't confuse outer (`out/.denet/`) vs inner (`<output_dir>/*.inner_denet.jsonl`) profiling.

3. **Module outputs must match benchmark.yaml**: Check the `outputs` section of each stage to know what files are expected.

4. **R and Python modules have different output conventions**: R methods save `.results.rds`, Python methods currently save `.access.tsv` instead.

5. **The omnibenchmark framework generates Snakefile**: Don't edit `out/Snakefile` directly, it's regenerated. Changes go in `benchmark.yaml`.

6. **Resource limits are enforced**: podman cgroup limits (CPU, memory) are set per module in `benchmark.yaml`.

7. **Development workflow**: Use `dev` entrypoints or run scripts directly during development. Only use `profile` entrypoints for full profiled runs.
